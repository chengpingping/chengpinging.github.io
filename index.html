github的使用

Posted on 2019-09-10

什么是GitHub

github是一个面向开源及私有的软件项目托管平台，只支持git作为唯一的版本库进行托管。
之前使用hexo的框架为自己搭建了这个博客，使用GitHub托管了这个项目，使得别人能够访问到自己的博客。

分布式管理系统

集中式和分布式的区别：集中式中央服务器挂掉不能工作，分布式没有影响
之前使用过SVN就是集中式版本管理系统。

怎么将项目推送到github（基本项目发布操作）

1.在你的github网站中新建一个仓库（Repositories），填写相关数据；
2.复制仓库地址，选择HTTPS,例如:https://github.com/chengpingping/github.git；
3.在你已经创建的项目目录下，在路径栏中输入:cmd，进入dos界面;
4.首先，你有创建一个git的版本，在dos框中输入：git init;
5.此时你可以查看工作区有没有新的文件，也就是可以推送的文件：git statue;
6.添加你要推送的项目到暂存区:有两种方法;
添加某个文件：git add 文件名;
添加所有文件：git add .;
7.将暂存区的文件提交到版本记录中：git commit -m "提示信息";
8.连接你在开头建好的仓库：git remote add origin 仓库地址;
9.将版本库中的文件推送到远程仓库：git push origin master;
(到这里，你基本上就可以将项目推送到github平台上了);

工作区、版本库和暂存区

编辑文件都是在工作区

git add是把工作区的修改放入暂存区；
git commit是把暂存的区修改一次性做一次版本记录。

管理修改

git commit只会把暂存区的修改提交到版本记录中。

撤销修改

1.直接丢弃工作区的改动:

git checkout --文件

2.修改已经添加到暂存区，但未commit:

a. git resert HEAD 文件
b. git checkout --文件

3.已经commit

分支的管理

基本命令

1.查看分支

2.创建分支

git branch <name>

4.切换分支

5.创建切换分支

6.合并某分支到当前分支

8.删除分支

分支管理策略

1.合并的适合如果可以使用快速合并;
2.禁止快速合并--no-ff.

Bug分支

1.git stash
2.切换到bug所在分支，并创建切换到一个临时分支，修复bug
3.修复完bug之后，切换回bug所在的分支并合并临时分支的内容，合并使用no-ff模式
4.删除临时分支
5.切换回工作分支
6.git stash pop

4.xss(cross site scripting)跨站脚本攻击

（我在笔试遇到两次）

什么是xss

1.是为了不和叠层样式表css（cascading style sheet)混淆，才缩写为xss
2.xss是一种常见的web应用中的计算机安全漏洞，他允许恶意的web用户将代码植入到提供给其他用户使用的页面中。

主要的危害

1.盗取用户的账号
2.控制企业数据：读取、篡改、添加、删除
3.盗取企业重要的具有价值的资料
4.非法转账
5.强制发送邮件
6.网站挂马
7.控制受害者网站向其他网站发起攻击

攻击方式

1.反射型，也叫非持久型
反射型xss代码首先出现在url中，然后服务端解析，最后浏览器解析后xss代码才能够攻击。

2.存储型，也叫持久型
主要是将xss代码发送到服务器，然后下一次请求的时候就不用带上xss代码了。
常用来窃取用户信息
典型的例子：
1.留言板xss。恶意用户留言的是js脚本，当留言被别人查看时，会被浏览器解析出来。
2.利用存储型的xss生成一些图片和文字，当其他用户去点击的时候就被xss攻击了，你的信息会被窃取或者被诱导至一个钓鱼网站。
3.劫持流量恶意跳转，用户打开网站之后会默认跳到指定网站。

高效记忆单词（词根、词缀法）

Posted on 2019-07-11

记单词对于我来说太痛苦了，死记一切东西对于我来说都是很难得事情。最近学习了一种新的单词记忆方法：词根词缀记忆法。

音记法（从前）

现在终于不用再百词斩上干记单词了，以前记忆单词的方法都是音记（利用读音记忆），超级慢还特别容易忘记，如果单词读音和单词意思之间是没有任何联系的，每记一个单词就必须在大脑中开辟一个空间记忆它的意思。

有的单词读音没有忘记（读音和拼音有相似之处，容易上手），意思已经完全不记得。

相似的单词，造成记忆混淆，有的单词比如：expect和except，我知道他们一个是‘期待’一个是‘除...外’但是总是搞混，如果使用词根词缀法就很好记忆。（ex：出、外面，pect:出现，cept:拿）

词根、词缀记忆法

词根、词缀

词根：决定了单词的意思
词缀：前缀决定单词的方向、大小等；后缀决定单词的词性。

ab-/abs-表示否定

abuse 滥用

ab:表示否定，负面的
use:使用

abduct 绑架

duct/duce:牵引
  • seduce 诱惑

    se:sex 色

  • introduce 接受

    intro：向内的

    in:向内、否定

  • education 教育

    ation:名词后缀

    aberrant:违反

    err:erro 错误
    ant:adj.(后缀)/n.
  • assisant 助理

    assist:帮助

  • accountant 会计

    accont:数

  • important 重要的

    import:进口

    port:搬运

    portable:便携的

    able:能够
    por：孔
    porous:多孔的
    ous:adj(后缀),多

    famous:有名的

hexo环境搭建和项目新建

Posted on 2019-07-11

hexo是一个博客框架，使用Markdown语法编写内容，在官方网站中提供了很多的主题，可以美化你的博客界面。
官网：https://hexo.io/zh-cn/ （中文）

安装Git和Node.js

Git和nodejs为hexo框架提供环境
Git下载地址：https://git-scm.com/downloads
NodeJS下载地址：http://nodejs.cn/download/
根据自己电脑情况，下载对应的版本。安装步骤简单，没有什么特别的注意事项，如果不想安装到C盘可以换到其他盘！
检查是否安装成功：
打开dos界面：win+R/直接搜索‘DOS’或者‘命令’


Git安装成功后，鼠标右键会有Git GUI Here和Git Bash Here两个选项

安装hexo

右键–>点击Git Bash Here，会弹出一个窗口，然后输入:npm install -g hexo-cli，等待一会。。。

开始新建自己的博客

1.hexo安装成功后，选择/新建一个自己放博客项目的文件夹；
2.右键右键–>点击Git Bash Here，输入：hexo init(初始化自己的环境)，可能会等待一会儿，也可能会失败（外网），成功后会在当前目录下会出现一些文件和文件夹；
3.输入：hexo generate/hexo g(生成静态界面)
4.输入：hexo server/hexo s(启动本地服务器)，此时你就可去访问你的第一个博客网站了：浏览器输入 http://localhost:4000 ；
5.输入：hexo new （这里输本篇博客的名字），新建自己的博客内容，会在：当前目录\source_posts 目录下生成一个后缀为.md的文件（Markdown文件）
6.编辑文件，可以用任意的文件编辑器编辑文档内容，比如：VSCode。Markdown是一种轻量级语言， 编写的文档可以导出 HTML 、Word、图像、PDF、Epub 等多种格式的文档。Markdown教程：https://www.runoob.com/markdown/md-tutorial.html
7.保存文档刷新页面，你就可以看见自己的博客内容。

将自己的博客发布到云服务器上

github是一个基于git的代码托管平台，付费用户可以建私人仓库，我们一般的免费用户只能使用公共仓库，也就是代码要公开，对于学习者这就够了。放在托管平台上的好处是任何人都可以通过网络访问到你的网站。

在GitHub上申请一个账号

官网：https://github.com/
注册成功登陆以后，你就可以新建一个仓库：Create a New Repository。（填上所有信息点击creat）
注意命名：GitHub账户名.github.io（严格按照此格式）

将自己的博客发布到GitHub上

在项目法人更目录下有一个_config.yml文件,修改Deployment下面的repo:https://GitHub账户名.github.io (这里填仓库地址)

上传代码到GitHub

1.安装：在Git窗口中输入 npm install hexo-deployer-git –save
2.执行hexo deploy/hexo d(输入自己的GitHub账号和密码)，你可以看见你的文件正在上传。
3.访问自己的仓库，浏览器中输入： https://GitHub账户名.github.io，就可成功访问自己的博客了。
