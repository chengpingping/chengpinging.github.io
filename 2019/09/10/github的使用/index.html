github的使用

Posted on 2019-09-10

什么是GitHub

github是一个面向开源及私有的软件项目托管平台，只支持git作为唯一的版本库进行托管。
之前使用hexo的框架为自己搭建了这个博客，使用GitHub托管了这个项目，使得别人能够访问到自己的博客。

分布式管理系统

集中式和分布式的区别：集中式中央服务器挂掉不能工作，分布式没有影响
之前使用过SVN就是集中式版本管理系统。

怎么将项目推送到github（基本项目发布操作）

1.在你的github网站中新建一个仓库（Repositories），填写相关数据；
2.复制仓库地址，选择HTTPS,例如:https://github.com/chengpingping/github.git；
3.在你已经创建的项目目录下，在路径栏中输入:cmd，进入dos界面;
4.首先，你有创建一个git的版本，在dos框中输入：git init;
5.此时你可以查看工作区有没有新的文件，也就是可以推送的文件：git statue;
6.添加你要推送的项目到暂存区:有两种方法;
添加某个文件：git add 文件名;
添加所有文件：git add .;
7.将暂存区的文件提交到版本记录中：git commit -m "提示信息";
8.连接你在开头建好的仓库：git remote add origin 仓库地址;
9.将版本库中的文件推送到远程仓库：git push origin master;
(到这里，你基本上就可以将项目推送到github平台上了);

基本操作（比较详细）

创建版本库

git init

生成.git目录

版本创建

git add 文件或目录
git commit -m '文件说明信息'

查看版本记录

git log

版本回退

git reset --hard HEAD^

git resert --hard 版本序列号

HEAD：指向当前版本

查看操作记录

git reflog

工作区、版本库和暂存区

编辑文件都是在工作区

git add是把工作区的修改放入暂存区；
git commit是把暂存的区修改一次性做一次版本记录。

管理修改

git commit只会把暂存区的修改提交到版本记录中。

撤销修改

1.直接丢弃工作区的改动:

git checkout --文件

2.修改已经添加到暂存区，但未commit:

a. git resert HEAD 文件
b. git checkout --文件

3.已经commit
使用版本回退操作

对比文件的不同

对比工作区和版本库某个文件

git diff HEAD --文件

对比两个版本中的文件

git diff HEAD HEAD^ --code.txt

删除文件

rm 文件

git rm 文件

git commit

GitHub网站的使用

创建仓库

new repository

添加ssh账户

1.点击头像——>settings——>ssh
2.生成电脑的ssh公钥
查看

vi .gitconfig

ssh

ssh-keygen -t rsa -C '邮箱‘
在.ssh下的id_rsa.pub文件中是新生程的密钥，加入到GitHub中

克隆项目

git clone ssh地址

推送代码

git push origin 分支名称

跟踪远程

git branch --set-upstream-to=origin/分支 分支

拉取代码

git pull origin 分支

分支的管理

基本命令

1.查看分支

git branch

2.创建分支

git branch <name>

4.切换分支

git checkout <name>

5.创建切换分支

git checkout -b <name>

6.合并某分支到当前分支

git merge <name>

禁用快速合并

git merge --no-ff <name>

8.删除分支

git branch -d <name>

分支冲突

分支有了新的记录并且修改的是同一个文件

分支管理策略

1.合并的适合如果可以使用快速合并;
2.禁止快速合并--no-ff.

Bug分支

1.git stash
2.切换到bug所在分支，并创建切换到一个临时分支，修复bug
3.修复完bug之后，切换回bug所在的分支并合并临时分支的内容，合并使用no-ff模式
4.删除临时分支
5.切换回工作分支
6.git stash pop

Master:用于保存发布的项目代码
Dev:保存开发过程中的代码
